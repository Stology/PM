# 합의_0717_자료수정 — 대기열 자료 메타데이터 수정

## 결정

- `수정하기`는 검토 결과를 다시 바꾸는 기능이 아니라, 업로더가 본인 자료의 **자료 제목**과 **자료 설명**을 수정하는 기능으로 정의한다.
- 문서와 화면의 필드명은 기존 용어인 `자료 설명`을 유지한다.
- 자료 수정은 해당 자료가 `UPL001`의 대기 중인 자료 영역에 존재하는 동안만 가능하다.
- 파일, 직접 입력 본문, 자동 귀속 주차, 업로더, AI 추출 결과는 수정 대상이 아니다.
- `검토 마치기`로 본인 검토를 제출한 뒤에는 승인/반려 결과를 다시 수정할 수 없다.
- 확정 완료 자료와 대기열에서 제거된 자료에는 자료 수정 기능을 제공하지 않는다.

## 화면 계약

- 추가 화면은 독립 페이지가 아니라 `UPL001-0200 자료 수정 모달` 파생 상태로 관리한다.
- 대기열에서 본인이 업로드한 자료 행에만 `자료 수정` 보조 버튼을 표시한다.
- `자료 수정` 클릭 시 컴팩트 모달을 열고 현재 자료 제목과 자료 설명을 미리 채운다.
- 모달 구성은 제목, 안내 문구, 자료 제목 필수 입력, 자료 설명 선택 입력, `취소`, `저장`으로 제한한다.
- 저장 성공 시 모달을 닫고 대기열 행의 제목과 자료 설명 데이터를 갱신한다.
- 제목이 비어 있으면 저장 버튼을 비활성화한다.
- 저장 중 자료가 대기열에서 제거되었거나 권한이 달라진 경우 저장하지 않고 최신 목록을 다시 확인하도록 안내한다.

## 검토 계약 변경

- `REV001` 진입 조건에서 `수정하기` 상태를 제거하고 `검토 필요` 상태만 유지한다.
- 후보별 승인/반려는 `검토 마치기` 전까지만 변경 가능하다.
- `검토 마치기` 제출 후 승인 기준 미달이면 다른 멤버의 검토를 기다리는 대기 상태로 유지하며, 제출한 사용자의 재진입·재수정 CTA는 제공하지 않는다.
- 동시성으로 확정된 후보와 확정 완료 자료는 기존과 같이 변경을 차단한다.

## 홈 계약 변경

- `HOM001-0400 자료 상세`의 `검토` 집계와 필터에는 `검토 필요`만 포함한다.
- 홈 자료 상세에서 검토 결과 재수정을 의미하던 `수정하기` 상태와 CTA를 제거한다.
- 자료 제목·자료 설명 수정은 `UPL001` 대기열에서만 제공한다.

## 근거 확인 결과

- Notion `기능 명세서`의 `STD-UP-LIST-01`, `HOME-ACT-01`, `REV-ENT-01`, `REV-SUB-01`에 기존 검토 재진입 계약이 남아 있었다.
- Figma `Stology_PM_Wireframe`의 `05-02. 대기 자료 상태 UPL001`, `06-03. 검토 완료 분기 REV001`, `HOM001-0400 자료 상세`에 같은 계약이 남아 있었다.
- 로컬 `docs/화면설계서.md`, `04-study-upload.md`, `09-review-screen.md`, `02-home.md`에도 같은 계약이 남아 있었다.
- 신규 모달 시안은 Superpowers Visual Companion에서 비교한 A안 `컴팩트 모달`로 확정했다. 확정안은 Figma `Stology_PM_Wireframe`에 `UPL001-0200 자료 수정 모달` 파생 프레임으로 반영한다.

## 검증 기준

- 업로드 화면에는 주차 선택 입력이 없어야 한다.
- 본인 대기 자료에서만 `자료 수정` 버튼이 보여야 한다.
- 모달에는 자료 제목과 자료 설명만 있어야 한다.
- 검토 화면과 홈 자료 상세에는 검토 결과 재수정 기능이 없어야 한다.
- Notion과 로컬 문서의 기능 ID, 화면 ID, 용어가 일치해야 한다.
