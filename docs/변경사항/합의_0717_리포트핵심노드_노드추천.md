# 합의 0717 — 리포트 핵심 노드와 노드 추천 정보 구조

## 배경

- `08. 주차별 리포트 RPT001`의 `노드 추천 · 이어서 해보기` 섹션에서 왼쪽 추천 노드별 설명과 오른쪽 AI 요약이 같은 내용을 반복했다.
- `이번 주 핵심 노드`가 어떤 데이터를 기준으로 구성되는지 기능 명세가 누락되어 있었다.

## 합의

1. 리포트 섹션 2의 이름은 `노드 추천`으로 정리한다.
2. 추천 결과는 `추천 유형(놓친/심화/연결) + 노드명 + 개별 추천 사유`로 구성한 전체 폭 단일 목록으로 표시한다.
3. 추천 노드 목록을 다시 문장으로 요약하던 별도 `이어서 해보기` 패널은 제거한다.
4. `이번 주 핵심 노드`는 리포트 주차와 온톨로지 `Concept.week` 값이 같은 노드를 조회해 구성한다.
5. 핵심 노드는 활성/비활성, 자료 연결 여부와 무관하게 주차 값이 일치하면 포함한다.
6. `week = 0`인 공통·선행 노드는 특정 주차 핵심 노드 목록에서 제외한다.
7. 핵심 노드 행에는 노드명, 현재 학습 상태, 관련 자료 수를 표시한다.
8. 생성 완료된 리포트는 생성 시점의 핵심 노드 목록과 표시 값을 스냅샷으로 유지하며 이후 활동으로 재계산하지 않는다.
9. `신규 활성화/보강 노드`는 해당 주차의 활동 결과이고, `이번 주 핵심 노드`는 템플릿에 정의된 학습 범위이므로 두 집합이 다를 수 있다.

## 빈 상태

- 핵심 노드 0개: `이번 주 학습 예정 노드가 없습니다.`
- 추천 노드 0개: `추천할 노드가 없습니다.`
