## 통합 원칙

1. Figma 프레임의 `화면 ID` 값은 Notion 기준 화면 ID 또는 상태 ID만 사용한다.
2. 과거 임시 ID(`AUTH001`, `HOME001`, `MOD_STUDY_CREATE` 등)는 제거한다.
3. 기능 ID는 `화면설명(Description)`의 `관련 기능 ID`로만 표기한다.
4. 하나의 Figma 화면이 4개 이상 기능을 설명하거나 예외/검증/권한 상태가 겹치면 별도 상태 프레임으로 분리한다.
5. 파생 화면은 부모 화면 ID 뒤에 상태 suffix를 붙인다. 예: `HOM001-0101`, `REV001-0200`.
6. 종료 스터디 쓰기 차단과 읽기 전용 정책의 소유 기능은 `STD-END-02`로 통일하고, 각 탭 프레임에는 `관련 기능 ID`로 연결한다.
7. 로그아웃은 공통 프로필 메뉴/글로벌 상태에 속하므로 `AUTH-OUT-01`을 `COMMON` expected 기능 ID에 포함한다.

## Figma 배치 원칙

1. 실제 서비스 화면은 기능 행의 상단에 배치한다.
2. 기능 묶음의 첫 번째 와이어프레임 상단에는 `기능: ...` 형식의 소제목을 붙인다.
3. 사용자에게 노출되는 독립 화면이 아니라 예외/검증/권한 정책만 설명하는 프레임은 웹 화면설계 레이아웃을 적용하지 않는다.
4. 예외정책 프레임은 `1540 x 580` 독립 정책 템플릿으로 제작하고, 상단 메타데이터 표, `Web 1920x1080` 캔버스, `화면설명(Description)` 테이블, 사용자 플로우 박스를 넣지 않는다.
5. 예외정책 프레임에는 `예외정책 템플릿 / 실제 화면 아님` 배지, `연결 와이어프레임` 카드, 정책 요약, 3개 정책 행만 둔다.
6. 정책 행은 번호, 정책명, 처리 기준을 한 줄 구조로 정리한다.
7. 예외정책 프레임은 관련 기준 와이어프레임의 하단 행에 배치한다. 동일 기준 화면에 여러 예외정책이 있으면 첫 정책은 같은 x좌표에, 추가 정책은 오른쪽으로 이어 배치한다.
8. 기능 묶음 사이에는 실제 화면 행과 예외정책 행이 섞여 보이지 않도록 충분한 세로 여백을 둔다.

## 정책 보강

1. `AUTH-LGN-01`: 인증 실패는 로그인 화면 유지 + toast, 비로그인 보호 URL 접근은 `LGN001`로 리다이렉트하고 원래 목적지를 보존한다.
2. `AUTH-RDR-01`: 유효하지 않은 초대 토큰과 종료된 스터디 초대 링크는 초대 오류 상태로 분기한다.
3. `INV001`: 초대 수락 실패는 toast 후 현재 화면 유지, 거절/취소/닫기 버튼은 제공하지 않는다.
4. `HOME-ACT-01`: 활동 로그 클릭 시 관련 출처로 이동하며, 삭제/권한 없음은 toast 후 홈 화면을 유지한다.
5. `HOME-CRT-01`: 스터디명 20자, 설명 200자, 날짜 `YYYY-MM-DD`, 과거 날짜 허용, 필수값 누락 시 `생성하기` disabled를 명시한다.
6. `HOME-INV-01`: 생성 직후 닫기는 홈으로, 설정 진입 닫기는 스터디 페이지로 복귀한다.
7. `SRC-POP-01`: X/외부 클릭으로 닫고, 닫은 뒤 지식 구조 탭의 선택 노드/필터 상태를 보존한다.
8. `REV-*`: 비멤버 접근 차단, 선택 없음 disabled, 확정 후보 제외, 제출 실패 toast 후 화면 유지, 제출 후 `수정하기`/확정 상태 전환을 명시한다.
9. `REV-PASS-01`: 승인 기준 충족 시 활성화, 1명이라도 반려 시 미활성, 기준 미달은 pending으로 분기한다.
10. `QNA-*`: 페이지네이션, 이미지 첨부 배지/본문 이미지/인라인 붙여넣기, 제목 50자/본문 1000자/필수값 disabled, 종료 스터디 작성 UI 숨김, 본인/타인 권한 차이, 이미 삭제된 항목 reload 안내를 명시한다.
11. `RPT001`은 주차별 기록 하위가 아닌 독립 탭이므로 Figma 번호를 `08.*`, 질문함은 `09.*`, 공통 상태는 `10.*`로 정리한다.
