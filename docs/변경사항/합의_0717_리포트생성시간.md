# 합의_0717_리포트생성시간 — 주차 종료 및 리포트 자동 생성 일정

## 결정

- 스터디 생성 시 `시작일`을 필수로 입력한다.
- 시작일을 기준으로 7일 단위 주차를 계산하며, 시작 요일의 전날을 고정된 주차 종료 요일로 삼는다.
- 기본 주차 종료 시각과 리포트 자동 생성 시각은 해당 종료 요일의 `24:00`이다.
- `24:00`은 화면 표시이며 시스템에서는 다음 날 `00:00`으로 처리한다.
- 기준 시간대는 `Asia/Seoul`이다.
- 주차가 종료되면 해당 주차 리포트를 자동 생성하며 사용자 수동 생성 액션은 제공하지 않는다.

## 생성 계약

- 스터디 생성 모달에는 시작일 선택 결과로 계산된 `매주 {종료 요일} 24:00에 주차 종료 및 리포트 자동 생성` 안내를 표시한다.
- 예: 시작일이 수요일이면 주차 범위는 수요일 00:00부터 다음 화요일 24:00까지이며, 기본 리포트 생성 일정은 매주 화요일 24:00이다.
- 생성 후 시작일과 종료 요일은 수정할 수 없다.

## 시간 수정 계약

- 진행 중인 스터디의 스터디장만 ⚙ 설정에서 `리포트 생성 시간`을 수정할 수 있다.
- 종료 요일은 시작일로 확정된 값을 읽기 전용으로 표시하고 시각만 수정한다.
- 스터디 시작 전 변경한 시각은 1주차부터 적용한다.
- 스터디 시작 후 변경한 시각은 현재 주차의 종료 시각을 바꾸지 않고 다음 주차부터 적용한다.
- 적용 대기 중 다시 수정하면 마지막으로 저장한 시각으로 예약값을 교체한다.
- 이미 종료된 주차의 기간과 생성된 리포트는 재계산하거나 재생성하지 않는다.
- 설정 모달에는 현재 적용 일정, 적용 예정 일정, 다음 리포트 생성 예정 일시를 표시한다.

## 화면·기능 계약

- `HOM001-0100`: 시작일 입력 아래 기본 리포트 생성 일정 안내를 표시한다.
- `MGT001`: 설정 드롭다운에 `리포트 생성 시간` 항목을 추가한다.
- `MGT001-0200`: 리포트 생성 시간 수정 모달을 추가한다.
- `STD-MGT-03`: 리포트 생성 시간 조회·수정 및 적용 시점 정책을 소유한다.
- `RPT-WEEK-01`: 설정된 주차 종료 시각을 기준으로 완료 주차와 미완료 주차를 구분한다.

## 검증 기준

- 시작일의 전날과 기본 `24:00`이 생성 화면, 설정 화면, 리포트 명세에서 동일하게 설명되어야 한다.
- 스터디원과 종료된 스터디에는 시간 수정 진입점이 없어야 한다.
- 시작 후 변경값은 다음 주차부터 적용되고 완료 리포트는 변하지 않아야 한다.
- Notion 기능 명세서, Figma Description, 로컬 화면설계서의 화면 ID와 기능 ID가 일치해야 한다.
