# 합의_0615 — 지식 구조 노드 상세 패널 관계 표시 방식 변경

## 합의 일자

- 2026-06-15

## 변경 대상

- 화면: `STD001` 지식 구조
- 기능: `STD-KNW-02` 활성 노드 상세 패널, `STD-KNW-03` 비활성 노드 상세 패널
- 산출물: Notion 기능 명세서, `docs/화면설계서.md`, `docs/wire_frame_requirements/03-study-knowledge.md`, Figma `Stology_PM_Wireframe`

## 합의 내용

1. 노드 상세 패널의 관계 영역은 4개 관계를 한 번에 리스트업하지 않는다.
2. `기반 개념`, `맥락 개념`, `확장 개념`, `대조 개념`을 토글/세그먼트 컨트롤로 제공한다.
3. 사용자가 관계 토글을 선택하면 해당 관계 유형의 연결 노드만 하단 목록에 표시한다.
4. 기본 선택값은 연결 노드가 있는 첫 번째 관계 유형이며, 연결 노드가 모두 없으면 `기반 개념`을 기본 선택한다.
5. 선택된 관계에 연결 노드가 없으면 빈 상태 문구를 표시한다.
