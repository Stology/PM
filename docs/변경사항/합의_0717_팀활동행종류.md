## 변경 대상

- 화면: `HOM001`, `HOM001-0300`
- 기능 ID: `HOME-ACT-01`
- 산출물: Notion `기능 명세서`, Figma `Stology_PM_Wireframe`, `docs/화면설계서.md`, `docs/wire_frame_requirements/02-home.md`, `docs/wire_frame_requirements/00-index.md`

## 변경 배경

- 팀원의 자료 업로드는 AI 추출 완료 후 `내 할 일 > 자료`의 검토 대상으로 이어져 홈에서 같은 자료가 중복 노출될 수 있다.
- `AI 후보 추출 완료`는 팀원의 직접 활동이 아니라 시스템 처리 결과이며, `내 할 일 > 자료`의 `검토 필요`와 의미가 겹친다.
- `AI 후보 추출 완료`와 `지식 구조 반영`은 동일 자료 처리 과정의 중간·최종 이벤트이므로 팀 활동에는 공유 가치가 높은 최종 결과만 남긴다.

## 확정 계약

팀 활동에는 다음 두 종류의 행만 표시한다.

| 이벤트 | 표시 조건 | 이동 위치 |
|---|---|---|
| 지식 구조 반영 | 승인된 개념 또는 관계가 지식 구조에 반영됨 | 해당 스터디의 지식 구조 탭 `STD001` |
| 다른 팀원의 질문에 등록된 답글 | 본인 질문이 아닌 다른 팀원의 질문에 답글이 등록됨 | 해당 질문이 펼쳐진 `QNA001` |

다음 이벤트는 팀 활동에서 제외한다.

- 팀원의 자료 업로드
- AI 후보 추출 완료
- 팀원의 새 질문
- 본인 질문에 등록된 새 답글
- 후보 승인/반려 개별 이력
- 리포트 생성

## 표시 구조

- 팀 활동은 `HOM001` 우측의 기존 패널과 행 UI 안에서 제공하며 별도 상세 모달을 열지 않는다.
- 스터디 단위 선택 컨트롤은 현재 UI의 `전체 스터디` 드롭다운을 그대로 유지한다.
- 기본값은 `전체 스터디`이고, 특정 진행 스터디를 선택하면 해당 스터디의 행만 같은 패널 안에 유지해 표시한다.
- 두 행 종류를 유형별 섹션이나 탭으로 나누지 않고 이벤트 발생 시간 최신순 단일 목록으로 표시한다.
- 목록 영역만 세로 스크롤하고 행 전체를 클릭 영역으로 사용한다.
- 각 행에는 유형 배지, 스터디명, 요약 문구, 대상명, 발생 시간을 표시한다.
- 필터 결과가 0건이면 `아직 팀 활동이 없습니다.`를 표시한다.

## 후속 문서 관계

- 본 합의는 `합의_0627_홈활동영역.md`와 `합의_0716_홈활동상세모달.md`의 팀 활동 표시 행 계약을 대체한다.
- 내 할 일의 자료·질문함·리포트 구성과 상세 모달은 변경하지 않는다.
- 팀 활동의 기존 인라인 패널, 스터디 단위 선택, 최신순 행 목록, 빈 상태와 권한 예외 정책은 그대로 유지한다.

## 검증 기준

- Notion, Figma, 로컬 문서 모두 팀 활동 행 종류를 `지식 구조 반영`, `다른 팀원의 질문에 등록된 답글` 두 개로만 설명해야 한다.
- 자료 업로드와 AI 후보 추출 완료를 팀 활동의 표시 행 또는 이동 대상으로 설명하는 문구가 없어야 한다.
- Figma HOM001 샘플 행과 Description의 이벤트 종류 및 이동 목적지가 일치해야 한다.
- 팀 활동을 상세 모달로 설명하거나 팀 활동 행 클릭 시 홈 파생 모달을 여는 흐름이 없어야 한다.
