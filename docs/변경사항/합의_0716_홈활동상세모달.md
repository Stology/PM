## 변경 배경

- 기존 홈의 `내 할 일`은 검토·답글·리포트 유형별 요약을 제공하지만, 같은 유형에 여러 대상이 생기면 홈 카드 하나에 대상명과 출처를 모두 표시하기 어렵다.
- 홈의 기존 3개 섹션 디자인은 유지하면서, 유형별 개별 대상을 상세 모달에서 확인하고 각 출처로 이동할 수 있어야 한다.
- 질문과 답글은 완료 태스크가 아니라 알림으로 관리하되, 사용자가 확인한 상태를 구분할 수 있어야 한다.
- 리포트 생성 시점은 스터디마다 다르므로, 참여 스터디별 최신 생성 리포트를 선택할 수 있어야 한다.
- 팀 활동은 유형별 구획보다 최근 흐름을 빠르게 파악하는 단일 피드 역할을 유지한다.

## HOM001 내 할 일 구조

1. 홈의 기존 `내 할 일` 영역과 3개 섹션 배치를 유지한다.
2. 섹션명은 `자료`, `질문함`, `리포트`로 정리한다.
3. 각 섹션을 클릭하면 해당 상세 모달을 연다.
4. 홈 요약 문구는 다음 형식을 사용한다.

| 섹션 | 하단 요약 | 우측 건수 |
|---|---|---|
| 자료 | `검토 N · 재업로드 N` | 검토·수정·재업로드 대상 합계 |
| 질문함 | `질문 N · 답글 N` | 읽지 않은 질문·답글 합계 |
| 리포트 | `N개 스터디의 최신 리포트` | 최신 생성 리포트가 있는 진행 스터디 수 |

## 자료 상세 모달 `HOM001-0400`

### 포함 대상

- 사용자가 검토해야 하는 `검토 필요` 자료
- 사용자가 다시 검토 화면에 진입해야 하는 `수정하기` 자료
- 사용자가 업로드했으나 AI 추출에 실패한 `재업로드 필요` 자료
- 홈 요약과 상세 필터에서는 `검토 필요`와 `수정하기`를 `검토`로 합산한다.

### 필터와 정렬

- 필터: `전체 N`, `검토 N`, `재업로드 필요 N`
- 기본값: `전체`
- `재업로드 필요` 자료를 먼저 표시한다.
- 같은 상태에서는 업로드 날짜 최신순으로 표시한다.

### 행 표시 정보

- 상태
- 자료명
- 스터디명
- 귀속 주차
- 업로더
- 업로드 날짜
- CTA: `검토하기`, `수정하기`, `재업로드`
- AI 후보 수와 후보명은 표시하지 않는다.

### 이동과 상태 처리

- `검토 필요`, `수정하기` 행은 해당 자료의 `REV001`로 이동한다.
- `재업로드 필요` 행은 해당 스터디의 `UPL001`로 이동한다.
- 재업로드 시 기존 파일·본문·제목을 복원하지 않고 사용자가 새로 입력한다.
- `합의_0716_업로드주차.md`에 따라 주차 입력은 제공하지 않으며, 새 자료는 등록 시점의 현재 진행 주차에 자동 귀속한다.
- 행 전체와 CTA 버튼은 같은 동작을 수행한다.
- 이동 시 모달을 닫고, 뒤로가기로 홈에 돌아와도 모달을 자동 재오픈하지 않는다.
- 추출 실패 자료는 실패 시점부터 1일간 유지하며, 기존 자동 삭제 시점에 홈과 모달에서도 제거한다.
- 재업로드 화면으로 이동한 것만으로는 기존 실패 항목을 제거하지 않는다.

### 빈 상태와 스크롤

- 0건이어도 홈 `자료` 섹션은 클릭할 수 있다.
- 모달 빈 상태 문구: `현재 확인할 자료가 없습니다.`
- 헤더와 필터는 고정하고 목록 영역만 세로 스크롤한다.
- 페이지네이션과 추가 로딩은 제공하지 않는다.

## 질문함 상세 모달 `HOM001-0500`

### 알림 포함 범위

- `새 질문`: 사용자가 참여 중인 진행 스터디에서 다른 팀원이 작성한 질문
- `새 답글`: 사용자가 작성한 질문에 다른 팀원이 작성한 답글
- 본인이 작성한 질문은 `새 질문` 알림에서 제외한다.
- 현재 질문함은 질문에 직접 답글이 달리는 단일 깊이 구조이며 대댓글은 지원하지 않는다.

### 필터와 목록 단위

- 필터: `전체 N`, `새 질문 N`, `새 답글 N`
- 기본값: `전체`
- 질문 이벤트 1개와 답글 이벤트 1개를 각각 1행으로 표시한다.
- 같은 질문에 답글 3개가 등록되면 같은 질문 제목의 답글 행 3개를 표시한다.
- 전체 목록은 이벤트 발생 시간 최신순으로 정렬한다.

### 행 표시 정보

- 유형 및 읽음 상태
- 질문 제목
- 스터디명
- 질문 또는 답글 작성자
- 작성일시
- CTA: `질문 보기`, `답글 보기`
- 답글 본문과 첨부 이미지는 표시하지 않는다.

### 읽음과 이동 정책

- 행 전체와 CTA 클릭 시 해당 질문이 펼쳐진 `QNA001`로 이동한다.
- 질문을 열면 해당 질문에 연결된 새 질문·새 답글 알림을 모두 읽음 처리한다.
- 읽은 행은 시각적 강조를 낮추고 모달에 유지한다.
- 홈의 `질문 N · 답글 N`과 우측 총 건수는 읽지 않은 알림만 집계한다.
- 모달의 필터 건수는 다음 정리 시점까지 남아 있는 읽음·미확인 행을 포함한다.
- 매일 한국 표준시(KST) 00시에 읽은 행만 제거한다.
- 읽지 않은 행은 날짜가 지나도 유지한다.

### 빈 상태와 스크롤

- 0건이어도 홈 `질문함` 섹션은 클릭할 수 있다.
- 모달 빈 상태 문구: `새로 등록된 질문이나 답글이 없습니다.`
- 헤더와 필터는 고정하고 목록 영역만 세로 스크롤한다.

## 리포트 상세 모달 `HOM001-0600`

### 목록 단위와 정렬

- 사용자가 참여 중인 진행 스터디를 1개당 1행으로 표시한다.
- 각 행은 해당 스터디에서 가장 최근 생성된 주차별 리포트 1개만 표시한다.
- 생성된 리포트가 있는 스터디를 먼저 표시하고, 생성일 최신순으로 정렬한다.
- 리포트가 없는 스터디는 그 뒤에 표시한다.

### 행 표시 정보

- 스터디명
- 최신 리포트명: 예) `4주차 커버리지 리포트`
- 생성일
- CTA: `리포트 보기`
- 스터디 제목 옆에는 별도 주차 배지를 표시하지 않는다.

### 이동과 상태 처리

- 행 전체와 CTA 클릭 시 해당 스터디·주차의 `RPT001`로 이동한다.
- 생성된 리포트가 없는 스터디는 `생성 전` 상태와 비활성 CTA를 표시한다.
- 0건이어도 홈 `리포트` 섹션은 클릭할 수 있다.
- 모달 빈 상태 문구: `참여 중인 스터디가 없습니다.`
- 목록 영역만 세로 스크롤한다.

## 팀 활동

### 표시 구조

- 팀 활동은 `HOM001` 우측의 기존 패널과 행 UI 안에서 제공하며 별도 상세 모달을 열지 않는다.
- 기존 `전체 스터디` 드롭다운을 스터디 단위 선택 컨트롤로 유지한다.
- 유형별 섹션, 그룹, 탭을 만들지 않는다.
- 기본값은 `전체 스터디`이며 특정 진행 스터디를 선택하면 해당 스터디의 행만 같은 패널 안에 유지해 최신순으로 표시한다.
- 목록 영역은 세로 스크롤한다.
- 각 행에는 요약 문구, 스터디명, 대상명, 발생 시간을 표시한다.

### 행 라우팅

| 이벤트 | 이동 |
|---|---|
| 지식 구조 반영 | 해당 스터디의 지식 구조 탭 |
| 다른 팀원의 질문에 등록된 답글 | 해당 질문이 펼쳐진 `QNA001` |

- 행 전체를 클릭 영역으로 사용한다.

### 질문함 중복 제외

- 팀원의 새 질문은 `질문함` 섹션에만 표시하고 팀 활동에서 제외한다.
- 내 질문에 달린 새 답글은 `질문함` 섹션에만 표시하고 팀 활동에서 제외한다.
- 다른 팀원의 질문에 달린 답글은 팀 활동에 유지한다.
- 팀원의 자료 업로드와 AI 후보 추출 완료는 `내 할 일 > 자료`와의 중복을 막기 위해 팀 활동에서 제외한다.
- 지식 구조 반영은 팀 활동에 유지한다.

## 공통 예외 처리

내 할 일 상세 모달의 행 또는 홈 우측 팀 활동 행을 클릭한 뒤 대상 삭제, 상태 변경, 권한 상실이 확인되면 다음 순서로 처리한다.

1. 페이지 이동을 차단한다.
2. 안내 토스트를 표시한다.
3. 현재 홈 또는 모달 상태를 유지한다.
4. 목록을 새로고침한다.
5. 유효하지 않은 행을 제거한다.
6. 홈 요약, 총 건수, 필터별 건수를 갱신한다.
