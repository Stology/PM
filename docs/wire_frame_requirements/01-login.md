# 01. 로그인 화면

> 와이어프레임 제작용 디자인 및 기능 요구사항

---

## 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 로그인 화면 |
| 화면 유형 | 풀스크린 페이지 |
| URL | `/login` |
| 권한 | 비로그인 (인증 없음) |
| 진입 경로 | 미인증 상태에서 모든 페이지 접근 시 / 로그아웃 직후 / 초대 링크 비로그인 진입 시 |

---

## 레이아웃 구조

```
┌──────────────────────────────────────────┐
│                                          │
│                                          │
│              [서비스 로고]                 │
│              "Stology"                   │
│                                          │
│       "당신의 활동이                       │
│        소중한 자산이 되도록"                │
│                                          │
│                                          │
│         ┌──────────────────┐             │
│         │ 카카오 로그인 버튼   │             │
│         └──────────────────┘             │
│                                          │
│                                          │
└──────────────────────────────────────────┘
```

- 중앙 정렬, 세로 배치
- 좌측 사이드바, 우측 상단 프로필 없음 (로그인 전)
- 단일 CTA로 시선 집중

---

## 화면 구성 요소

### 1. 서비스 로고
- **위치**: 상단 중앙
- **내용**: Stology 로고 + 텍스트
- **클릭 동작**: 없음

### 2. 슬로건
- **위치**: 로고 아래
- **내용**: "당신의 활동이 소중한 자산이 되도록"
- **스타일**: 부제 텍스트, 한 줄

### 3. 카카오 로그인 버튼
- **위치**: 슬로건 아래 중앙
- **내용**: 카카오 디자인 가이드 준수 (카카오 노란색 #FEE500)
- **레이블**: "카카오로 시작하기" 또는 "카카오 로그인"
- **클릭 동작**: 카카오 OAuth 외부 페이지로 이동

---

## 기능 요구사항

### 인증 흐름

```
[로그인 화면]
   ↓ 카카오 로그인 버튼 클릭
[카카오 OAuth 외부 페이지]
   ↓ 인증 완료
[홈 화면]
```

### 인증 후 라우팅

| 사용자 구분 | 도착 화면 |
|---|---|
| 신규 사용자 (첫 로그인) | 홈 화면 |
| 기존 사용자 | 홈 화면 |
| 초대 링크로 진입 중인 사용자 | 초대 수락 화면 `INV001` |
| 보호 URL에서 리다이렉트된 사용자 | 인증 성공 후 원래 목적지 |

신규/기존 사용자 구분 없이 동일하게 홈으로 진입한다.

초대 링크로 진입한 사용자는 카카오 로그인 성공 후 자동으로 스터디에 참여하지 않는다. 별도 초대 수락 화면 `INV001`에서 초대 정보를 확인하고 `참여하기`를 선택해야 스터디에 참여한다.

### 인증 정책

- 카카오 소셜 로그인 단일 방식
- 모든 페이지 로그인 필수 (비로그인 시 이 화면으로 리다이렉트)
- 이메일/비밀번호, 다른 소셜 로그인 미지원
- 보호 URL 접근으로 `LGN001`에 온 경우 원래 목적지 URL을 보존한다.
- 카카오 인증 성공 시 보존된 목적지가 있으면 해당 목적지로 이동하고, 없으면 기본 홈으로 이동한다.
- 보존된 목적지가 권한 없음/삭제됨/종료 후 쓰기 URL이면 해당 화면의 권한 또는 상태 오류 정책을 우선 적용한다.

## 인터랙션 명세

| 인터랙션 | 동작 |
|---|---|
| 카카오 로그인 버튼 클릭 | 카카오 OAuth 외부 페이지 이동 |
| 일반 카카오 인증 성공 | 홈 화면으로 라우팅 |
| 초대 링크 카카오 인증 성공 | 초대 수락 화면 `INV001`로 라우팅 |
| 보호 URL 카카오 인증 성공 | 보존된 목적지로 라우팅 |
| 카카오 인증 실패 | 로그인 화면 유지 + 카카오 기본 오류 또는 상단 토스트 표시 |

---

## 빈 상태 / 에러 상태

| 상태 | 처리 |
|---|---|
| 카카오 인증 실패 | 로그인 화면 유지 + 오류 안내 |
| 비로그인 보호 URL 접근 | `LGN001-0100` 상태로 목적지 보존 안내 후 로그인 |
| 네트워크 끊김 | MVP 미정의 |

---

## 디자인 가이드

### 시각 우선순위
1. 로고 + 슬로건 (상단)
2. 카카오 로그인 버튼 (메인 CTA)

### 톤
- 미니멀
- 단일 CTA에 시선 집중
- 추가 요소 (회원가입 링크, 약관 등) 없음

### 반응형
- 웹 전용 (모바일 미고려)

## 관련 화면

| 관련 화면 | 관계 |
|---|---|
| 홈 화면 | 로그인 성공 시 진입 |
| 초대 수락 화면 | 초대 링크 로그인 성공 시 진입 |
