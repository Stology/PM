# 10. 초대 수락 화면

> 와이어프레임 제작용 디자인 및 기능 요구사항

---

## 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 초대 수락 화면 |
| 화면 ID | `INV001` |
| 화면 유형 | 풀스크린 페이지 |
| URL | `/invite/{token}` 또는 로그인 후 초대 토큰 검증 결과 화면 |
| 권한 | 로그인 사용자 |
| 진입 경로 | 초대 링크로 진입한 사용자가 카카오 로그인에 성공 |
| 관련 기능 ID | `AUTH-RDR-01`, `INV-JOIN-01` |

---

## 레이아웃 구조

```
┌──────────────────────────────────────────┐
│                                          │
│              [서비스 로고]                 │
│                                          │
│          스터디 초대가 도착했습니다           │
│                                          │
│      ┌────────────────────────────┐      │
│      │ 스터디명                    │      │
│      │ 스터디장: 김스터디            │      │
│      │ 현재 멤버: 5명               │      │
│      └────────────────────────────┘      │
│                                          │
│              [참여하기]                  │
│                                          │
└──────────────────────────────────────────┘
```

- 로그인 화면과 같은 단순 중앙 정렬 구조를 사용한다.
- 홈 사이드바와 프로필은 표시하지 않는다.
- 거절/취소 버튼은 제공하지 않는다.

---

## 화면 구성 요소

### 1. 서비스 로고
- **위치**: 상단 중앙
- **내용**: Stology 로고 + 텍스트
- **클릭 동작**: 없음

### 2. 초대 안내 제목
- **내용**: "스터디 초대가 도착했습니다"
- **목적**: 사용자가 초대 링크 인증 후 별도 수락 단계에 진입했음을 명확히 알린다.

### 3. 초대 스터디 정보 카드
- **표시 정보**
  - 스터디명
  - 스터디장
  - 현재 멤버 수
- **미표시 정보**
  - 온톨로지 템플릿 정보는 표시하지 않는다.

### 4. 참여하기 버튼
- **위치**: 초대 스터디 정보 카드 아래
- **클릭 동작**: 스터디 멤버로 추가한 뒤 해당 스터디의 지식 구조 탭 `STD001`로 이동한다.

---

## 기능 요구사항

### 초대 링크 검증

- 초대 토큰은 `INV001` 진입 전에 검증한다.
- 유효하지 않은 초대 토큰은 `INV001-0100` 초대 링크 오류 상태로 표시한다.
- 종료된 스터디의 초대 링크는 `INV001-0100`에서 "이미 종료된 스터디 초대입니다"로 안내한다.
- 초대 링크 오류 상태에서는 스터디 참여 CTA를 표시하지 않고 홈으로 이동 CTA만 제공한다.

### 초대 수락 흐름

```
[초대 링크 진입]
   ↓ 비로그인 상태
[LGN001 로그인]
   ↓ 카카오 로그인 성공
[INV001 초대 수락 화면]
   ↓ 참여하기
[STD001 지식 구조 탭]
```

### 참여 처리

- `참여하기` 클릭 시 초대 토큰을 기준으로 사용자를 스터디 멤버에 추가한다.
- 참여 성공 시 별도 토스트 없이 해당 스터디 페이지의 지식 구조 탭으로 이동한다.
- 참여 처리 중에는 `참여하기` 버튼을 로딩/비활성 상태로 표시한다.
- 참여 실패 시 현재 화면을 유지하고 상단 토스트로 실패 사유를 안내한다.

### 액션 제약

- 거절, 취소, 닫기 버튼은 제공하지 않는다.
- 사용자가 참여하지 않으려면 브라우저 뒤로가기 또는 다른 경로 이동으로 화면을 벗어난다.

## 인터랙션 명세

| 인터랙션 | 동작 |
|---|---|
| 참여하기 클릭 | 스터디 멤버 추가 → `STD001` 지식 구조 탭으로 이동 |
| 참여 처리 중 재클릭 | 버튼 비활성 상태로 중복 요청 방지 |
| 참여 실패 | 화면 유지 + 상단 토스트 표시 |

---

## 빈 상태 / 에러 상태

| 상태 | 처리 |
|---|---|
| 초대 정보 로딩 중 | 중앙 로딩 상태 표시 |
| 초대 정보 조회 실패 | `INV001-0100` 초대 링크 오류 상태 표시 |
| 유효하지 않은 초대 토큰 | `INV001-0100` 초대 링크 오류 상태 표시 |
| 종료된 스터디 초대 링크 | `INV001-0100` 초대 링크 오류 상태 표시 |
| 참여 실패 | 상단 토스트 표시, `참여하기` 버튼 재활성 |

### 화면 ID 추적성

- `INV001`은 초대 정보 확인 및 참여하기 단일 화면이다.
- 참여 성공 후 이동 대상은 `STD001` 지식 구조 탭이며, `INV001` 화면 ID 안에 이동 대상 ID를 섞어 쓰지 않는다.
- 참여 실패 상태는 `INV001-0200`으로 분리해 `toast 후 화면 유지`, `거절/취소 버튼 없음`을 검증한다.

---

## 디자인 가이드

- 로그인 화면과 유사한 단일 CTA 구조를 사용한다.
- 스터디 정보 카드가 화면의 주 정보가 되도록 배치한다.
- 거절 버튼을 두지 않으므로 `참여하기` 버튼 외 시각적 CTA를 추가하지 않는다.
