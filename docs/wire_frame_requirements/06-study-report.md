## 화면 구성 요소

### 1. 주차 토글 영역
- 토글 형식 (주차별 기록 탭과 동일 패턴)
- 1주차 ~ 현재 주차 노출
- 한 번에 하나 선택

### 2. 섹션 1: 진행 상황 요약

#### 시각 정보 1: 활성화 노드 수
- 큰 숫자 또는 카드 형태
- "이번 주 활성화 노드 수: N개"

#### 시각 정보 2: 도넛 차트
- **신규 활성화 노드 수 vs 보강 노드 수** 비율
- 두 색상으로 구분
- 차트 옆에 범례 (라벨 + 숫자)
- 예: 신규 활성화 5개 / 보강 7개

#### 이번 주 핵심 노드
- 선택한 리포트 주차와 온톨로지 `Concept.week` 값이 같은 노드를 모두 리스트업한다.
- 예: 4주차 리포트에서는 `week = 4`인 Concept를 조회한다.
- 자료 연결 또는 활성화 여부와 무관하게 해당 주차로 정의된 노드를 포함한다.
- 각 행에는 노드명, 현재 학습 상태, 관련 자료 수를 표시한다.
- `신규 활성화/보강`은 해당 주차의 활동 결과이고, `이번 주 핵심 노드`는 온톨로지에 정의된 학습 범위이므로 서로 다른 집합일 수 있다.
- 노드는 온톨로지 템플릿의 저장 순서를 기본으로 표시한다.

#### AI 텍스트
- 이번 주 학습 내용 요약 (AI 자동 생성)
- 한 문단 분량
- 주차별 업로드된 자료 기반

### 3. 섹션 2: 노드 추천

- 추천 유형은 `놓친`, `심화`, `연결`로 구분한다.
- 각 행에는 추천 노드명, 추천 유형, 해당 노드를 추천하는 개별 사유를 표시한다.
- 추천 노드 목록과 같은 내용을 다시 문장으로 요약하던 별도 `이어서 해보기` 패널은 표시하지 않는다.
- 추천 결과는 전체 폭의 단일 목록으로 표시한다.

### 4. 섹션 3: 팀 통계

#### 누적 막대그래프 (Stacked Bar)
- 팀원별 막대 1개
- **자료 업로드 수와 질문 작성 수를 색상으로 누적 표시**
- 색상으로 두 항목 구분
- 막대 옆에 팀원 이름과 수치

#### 팀원별 카드 (막대그래프 아래)
- 각 팀원당 카드 1개
- 카드 안에 AI 짧은 피드백 코멘트

## 기능 요구사항

### 리포트 생성 시점
- 시작일의 전날을 고정된 주차 종료 요일로 사용한다.
- 기본 주차 종료 및 리포트 생성 시각은 해당 요일 `24:00`이다.
- 스터디장이 수정한 생성 시각이 있으면 해당 시각에 주차를 종료하고 자동 생성한다.
- 시작 후 변경한 시각은 현재 주차가 아닌 다음 주차부터 적용한다.
- 이미 종료된 주차의 기간과 생성된 리포트는 재계산하거나 재생성하지 않는다.
- 기준 시간대는 `Asia/Seoul`이며 `24:00`은 시스템에서 다음 날 `00:00`으로 처리한다.
- 사용자 수동 생성 액션 없음

### 리포트 단위
- 주차별 1개씩
- 누적 종합 리포트 없음

### 이번 주 핵심 노드 조회
- 조회 조건: `Concept.week = report.week`
- 조회 대상: 스터디 생성 시 선택한 온톨로지 템플릿의 전체 Concept
- 포함 조건: 활성/비활성, 자료 연결 유무와 무관하게 주차 값이 일치하면 포함
- 표시 값: 노드명, 현재 학습 상태, 관련 자료 수
- 스냅샷: 리포트 생성 시점의 목록과 표시 값을 저장하며, 생성 완료 리포트는 이후 활동으로 재계산하지 않는다.
- `week = 0`인 공통/선행 Concept는 특정 주차 핵심 노드 목록에 포함하지 않는다.

### 권한
- 스터디 멤버 누구나 조회 가능
- 권한별 표시 차이 없음

### 외부 내보내기
- 없음 (조회만)

### 점수 산식 (팀 통계 막대그래프)
- MVP는 **단순 카운트** (자료 업로드 수 + 질문 작성 수)
- 가중치 부여 등은 추후 결정

| 케이스 | 처리 |
|---|---|
| 미완료 주차 선택 | 설정된 주차 종료 시각 전이면 "이번 주차 스터디가 완료되지 않았습니다" 안내 메시지 |

| 추천 노드 0개 | `추천할 노드가 없습니다.` 표시하고 별도 `이어서 해보기` 패널은 노출하지 않음 |

## 디자인 가이드

### 섹션 구분
- 카드 형태로 섹션 분리
- 섹션 제목 명확히 표시

### 시각 표현
- **숫자**: 큰 사이즈 강조 (활성화 노드 수)
- **도넛 차트**: 두 색상 명확히 구분, 범례 노출
- **막대그래프**: 누적 막대 색상으로 자료/질문 구분
- **핵심 노드**: 노드명, 학습 상태, 관련 자료 수를 비교할 수 있는 표
- **노드 추천**: 추천 유형, 노드명, 개별 사유를 한 행에서 읽을 수 있는 전체 폭 목록

### 색상 정책
- 도넛 차트: 신규 vs 보강 두 색상 (예: 메인 컬러 + 보조 컬러)
- 막대그래프: 자료 vs 질문 두 색상

### 팀원 카드
- 카드 그리드 또는 세로 나열
- 카드 안에 프로필 사진 + 이름 + AI 코멘트
