# 02. 홈 화면

> 와이어프레임 제작용 디자인 및 기능 요구사항
> 파생 화면: 스터디 생성 모달, 초대 링크 생성 모달(생성 직후 진입 케이스), 활동 영역 예외, 자료 상세 모달, 질문함 상세 모달, 리포트 상세 모달

---

## 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 홈 화면 |
| 화면 유형 | 풀스크린 페이지 |
| URL | `/` 또는 `/home` |
| 권한 | 로그인 사용자 |
| 진입 경로 | 카카오 로그인 직후 / 사이드바 "홈" 클릭 / 초대 링크 모달 닫기 시 |

---

## 레이아웃 구조

```
┌─────────────────────────────────────────────────────────────┐
│  [사이드바]              │    [우측 상단 프로필]              │
│                          │                                   │
│  - 홈                    │   ┌─ 진입점 섹션 ──────────────┐   │
│  - 스터디                 │   │ 스터디 (가로 1열, 스크롤)    │   │
│    └ 스터디 A             │   │ [카드A][카드B][카드C][+카드] │   │
│    └ 스터디 B             │   └────────────────────────────┘   │
│    └ 지난 활동             │                                   │
│      └ 종료 스터디 X       │   ┌─ 활동 영역 ─────────────────┐  │
│                          │   │ [내 할 일][팀 활동 + 드롭다운] │   │
│                          │   └────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 영역별 비율 가이드
- 좌측 사이드바: 약 20% 너비, 항상 펼침
- 우측 상단 프로필: 우상단 고정
- 메인 영역: 진입점 섹션 + 활동 영역 (위/아래 분할)
- 활동 영역: 좌우 50:50 분할

---

## 화면 구성 요소

### 1. 좌측 사이드바

#### 구조
```
- 홈                          ← 현재 위치
- 스터디
  - [참여 중인 스터디 1]
  - [참여 중인 스터디 2]
  - 지난 활동
    - [종료된 스터디 1]
    - [종료된 스터디 2]
```

#### 규칙
- **항상 펼침 상태 유지** (펼침/접힘 토글 없음)
- 상위 메뉴("스터디")는 클릭 안 됨 (라벨 역할)
- 현재 페이지 시각 표시 (홈에 있을 때는 "홈" 강조)
- 종료된 스터디는 "지난 활동" 섹션에서 접근

### 2. 우측 상단 프로필
- 카카오 프로필 사진 (원형)
- 이름
- 닉네임 (선택)
- 스터디장 배지 등 별도 표시 없음

### 3. 진입점 섹션 — 스터디

#### 레이아웃
- 가로 1열, 가로 스크롤
- 카드 형태 나열
- 맨 우측에 **+ 카드** (항상 노출)

#### 일반 스터디 카드 구성
- 스터디명
- 현재 주차 (시작일 기준 자동 계산)
- NEW 배지 (최근 1일 이내 자료 업로드 있을 시)

#### + 카드 구성
- 시각적으로 구분되는 카드 (예: 점선 테두리, + 아이콘)
- 라벨: "스터디 생성" 또는 "+ 새 스터디"
- 진행 중 스터디 0개일 때도 항상 노출

#### 클릭 동작
| 클릭 위치 | 동작 |
|---|---|
| 일반 스터디 카드 | 해당 스터디 페이지로 이동 (`/study/{id}`) |
| + 카드 | 스터디 생성 모달 호출 |

#### 표시 범위
- 진행 중인 스터디만
- 종료된 스터디는 미표시 (사이드바 "지난 활동"에서 접근)

### 4. 활동 영역 (좌우 분할)

#### 좌측: 내 할 일
- 현재 홈 디자인의 3개 섹션을 유지하고, 각 섹션은 요약 정보와 상세 모달 진입점 역할을 한다.

| 섹션 | 홈 보조 문구 | 우측 숫자 | 클릭 결과 |
|---|---|---|---|
| 자료 | `검토 N · 재업로드 N` | 검토 + 재업로드 필요 합계 | 자료 상세 `HOM001-0400` |
| 질문함 | `질문 N · 답글 N` | 읽지 않은 새 질문 + 새 답글 합계 | 질문함 상세 `HOM001-0500` |
| 리포트 | `N개 스터디의 최신 리포트` | 최신 리포트가 생성된 진행 스터디 수 | 리포트 상세 `HOM001-0600` |

- 홈에는 개별 대상/출처를 직접 나열하지 않는다.
- 0건이어도 3개 섹션은 유지하며 상세 모달의 빈 상태를 확인할 수 있다.

#### 우측: 팀 활동
- `HOM001` 우측의 기존 패널과 행 UI를 유지하며 별도 상세 모달을 제공하지 않는다.
- 참여 중인 진행 스터디에서 본인을 제외한 팀원 활동을 유형 구분 없이 하나의 목록으로 최신순 표시한다.
- 우측 상단의 기존 스터디 선택 드롭다운으로 표시할 스터디 단위를 전환한다.
  - 기본값: "전체 스터디"
  - 옵션: "전체 스터디" + 참여 중인 진행 스터디 목록
  - 스터디를 1개만 참여 중이면 드롭다운은 숨김 처리한다.
- 목록 영역만 세로 스크롤하며 페이지네이션/무한 스크롤은 사용하지 않는다.
- 행 전체가 클릭 영역이며 이벤트 유형에 따라 해당 화면으로 이동한다.

| 이벤트 | 이동 위치 |
|---|---|
| 지식 구조 반영 | 해당 스터디 지식 구조 탭 `STD001` |
| 다른 팀원의 질문에 등록된 답글 | 해당 질문이 펼쳐진 `QNA001` |

- 제외 항목:
  - 팀원의 자료 업로드와 AI 후보 추출 완료는 `내 할 일 > 자료`와 중복되므로 제외한다.
  - 팀원의 새 질문은 질문함 상세의 `새 질문`에만 표시한다.
  - 본인 질문에 달린 새 답글은 질문함 상세의 `새 답글`에만 표시한다.
  - 후보 승인/반려 개별 이력은 팀 활동에서 제외한다.
  - 리포트 생성은 팀 활동에서 제외한다.

#### 활동 항목 공통 카드 구성
- 유형 배지: 지식 구조, 답글
- 스터디명
- 요약 문구
- 대상명: 자료명, 질문명, 노드명 등
- 발생 시간
- 행 우측 이동 표시

## 기능 요구사항

### 진입점 카드 동작

**일반 스터디 카드 클릭**
- 해당 스터디 페이지 진입 (기본 탭은 지식 구조 탭)

**NEW 배지 표시 규칙**
- 최근 1일 이내 자료 업로드가 있을 시 노출
- 본인 업로드 포함 여부: MVP 미정의 (스터디 단위로 판단)

**가로 스크롤**
- 카드 수가 화면 너비를 초과하면 가로 스크롤
- + 카드는 항상 맨 우측 (스크롤 시 따라옴 또는 끝에 위치)

### 스터디 생성 진입

**+ 카드 클릭 시**
- 스터디 생성 모달 호출 (아래 파생 화면 참조)

### 활동 영역 동작

- `내 할 일`의 각 섹션을 클릭하면 동일한 공통 모달 구조를 사용하는 상세 화면을 연다.
- 상세 모달은 제목/닫기, 필터, 목록 헤더를 고정하고 목록 영역만 세로 스크롤한다.
- `팀 활동`은 별도 상세 모달을 열지 않고, 선택한 스터디 값에 맞는 행을 HOM001 우측 기존 패널의 단일 목록에 최신순으로 유지한다.
- 연결 대상이 삭제·상태 변경되었거나 사용자가 권한을 잃으면 이동을 막고 토스트를 표시한다. 현재 홈/모달에 머문 채 목록을 새로고침하여 유효하지 않은 행을 제거하고 홈·필터 숫자를 다시 계산한다.
- 활동 영역 예외 상태는 `HOM001-0300`으로 분리한다.

# 파생 화면 1. 스터디 생성 모달

## 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 스터디 생성 모달 |
| 화면 유형 | 모달 (오버레이) |
| 진입 경로 | 홈 화면 진입점 카드 영역 맨 우측 + 카드 클릭 |
| 권한 | 로그인 사용자 누구나 |

## 레이아웃 구조

```
┌─────────────────────────────────────┐
│  스터디 생성             [X]         │
├─────────────────────────────────────┤
│                                     │
│  스터디 이름 *                       │
│  ┌─────────────────────────────┐    │
│  │                             │    │
│  └─────────────────────────────┘    │
│  (최대 20자)                         │
│                                     │
│  온톨로지 템플릿 *                    │
│  ┌─────────────────────────────┐    │
│  │ 템플릿 검색어 입력...        │    │
│  └─────────────────────────────┘    │
│  ┌─────────────────────────────┐    │
│  │ Spring Boot 로드맵           │    │
│  │ 백엔드 스터디 기본 템플릿      │    │
│  │ 업로드 사용자: Stology        │    │
│  └─────────────────────────────┘    │
│                                     │
│  시작일 *                            │
│  ┌─────────────────────────────┐    │
│  │ YYYY-MM-DD                  │    │
│  └─────────────────────────────┘    │
│  매주 {종료 요일} 24:00에 주차 종료 및  │
│  리포트 자동 생성                     │
│                                     │
│  설명                                │
│  ┌─────────────────────────────┐    │
│  │                             │    │
│  │                             │    │
│  └─────────────────────────────┘    │
│  (최대 200자, 선택)                  │
│                                     │
│              [생성하기]              │
└─────────────────────────────────────┘
```

## 입력 필드

| 필드 | 필수 | 검증 | UI |
|---|---|---|---|
| 스터디 이름 | 필수 | 최대 20자, 중복 허용 | 단일 라인 텍스트 입력 |
| 온톨로지 템플릿 | 필수 | 검색 결과에서 1개 선택 | 검색 입력 + 후보 목록 + 선택 결과 |
| 시작일 | 필수 | YYYY-MM-DD, 과거 날짜 허용 | 텍스트 입력 |
| 기본 리포트 생성 일정 | 자동 계산 | 시작일의 전날, 24:00, Asia/Seoul | 읽기 전용 안내 |
| 설명 | 선택 | 최대 200자 | 여러 줄 텍스트 입력 |

### 시작일·기본 리포트 일정

- 시작일을 기준으로 7일 단위 주차를 계산한다.
- 시작 요일의 전날을 고정된 주차 종료 요일로 삼고, 기본 종료 및 리포트 생성 시각은 해당 요일 `24:00`이다.
- 예: 시작일이 수요일이면 `매주 화요일 24:00`으로 안내한다.
- `24:00`은 화면 표현이며 시스템에서는 다음 날 `00:00`으로 처리한다.
- 기준 시간대는 `Asia/Seoul`이다.
- 생성 후 시작일과 종료 요일은 수정할 수 없고, 스터디장은 설정에서 시각만 수정할 수 있다.

### 검증/비활성 상태

- `생성하기` 버튼은 스터디 이름, 온톨로지 템플릿, 시작일이 모두 유효할 때만 활성화한다.
- 스터디 이름 20자 초과, 설명 200자 초과는 필드 아래 인라인 메시지로 표시한다.
- 시작일 형식이 `YYYY-MM-DD`가 아니면 필드 아래 인라인 메시지를 표시한다.
- 과거 날짜는 허용하므로 과거 날짜만으로는 오류 처리하지 않는다.
- 검증/disabled 상태는 `HOM001-0102`로 분리한다.

### 온톨로지 템플릿 검색/선택

- 주소 검색 UI와 유사하게 검색어 입력 후 후보 목록을 표시한다.
- 후보 항목에는 `템플릿명`, `간단 설명`, `업로드 사용자`를 표시한다.
- MVP에서 업로드 사용자는 모두 `Stology`로 표시한다.
- `업로드 사용자` 필드는 추후 C2C 템플릿 확장을 고려해 유지한다.
- 후보를 클릭하면 선택 상태로 고정하고, 선택된 템플릿명은 입력 영역에 표시한다.
- 검색 결과가 없으면 "검색 결과가 없습니다" 빈 상태를 표시한다.
- 템플릿 검색/선택 상태는 `HOM001-0101`로 분리한다.

## 액션

| 버튼 | 동작 | 활성화 조건 |

| X (닫기) | 모달 닫힘, 입력 내용 폐기 | 항상 활성 |

## 에러 처리

- 폼 검증 에러 (예: 글자 수 초과): 인라인 메시지 (필드 아래)

# 파생 화면 3. 자료 상세 모달

## 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 자료 상세 |
| 화면 ID | `HOM001-0400` |
| Figma 프레임 | `02-09. 자료 상세 HOM001-0400` |
| 화면 유형 | 모달 (오버레이) |
| 진입 경로 | 홈 `내 할 일`의 자료 섹션 클릭 |
| 관련 기능 ID | `HOME-ACT-01, COMMON-FDB-01` |

## 공통 구조

```
┌────────────────────────────────────────────────────────────┐
│ 자료 상세                                             [X] │
├────────────────────────────────────────────────────────────┤
│ [전체 N] [검토 N] [재업로드 필요 N]                        │
├────────────────────────────────────────────────────────────┤
│ 상태 │ 자료 제목 │ 스터디 │ 귀속 주차 │ 업로더 │ 업로드일 │ 액션 │
│ ... 세로 스크롤 목록 ...                                   │
└────────────────────────────────────────────────────────────┘
```

- 모달 헤더, 필터, 목록 헤더는 고정하고 목록 본문만 세로 스크롤한다.
- 필터 숫자는 현재 모달 목록을 기준으로 계산한다.
- `검토` 필터에는 `검토 필요` 상태만 포함한다.
- `재업로드 필요`에는 로그인 사용자가 업로드한 자료의 `추출 실패`만 포함한다.
- 행에는 AI 후보 수를 표시하지 않는다.

## 행 및 CTA 정책

| 상태 | 행 정보 | CTA | 결과 |
|---|---|---|---|
| 검토 필요 | 상태, 자료 제목, 스터디, 귀속 주차, 업로더, 업로드일 | `검토하기` | 해당 `REV001`로 이동 |
| 추출 실패 | 상태, 자료 제목, 스터디, 귀속 주차, 업로더, 업로드일 | `재업로드` | 빈 `UPL001`로 이동 |

- 재업로드 진입 시 기존 파일·본문·제목을 미리 채우지 않는다.
- `UPL001`에는 주차 선택/입력 UI를 두지 않으며 새 자료는 업로드 시점의 스터디 현재 주차에 자동 귀속한다.
- 추출 실패 행은 실패 시점부터 1일간 유지한 뒤 자동 삭제한다.
- 재업로드 화면으로 이동했다는 이유만으로 기존 추출 실패 행을 제거하지 않는다.
- 자료 제목·자료 설명 수정은 홈 모달이 아니라 해당 스터디 `UPL001`의 대기열에서만 제공한다.
- 선택 필터에 항목이 없으면 목록 본문에 `내용 없음`을 표시한다.

# 파생 화면 4. 질문함 상세 모달

## 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 질문함 상세 |
| 화면 ID | `HOM001-0500` |
| Figma 프레임 | `02-10. 질문함 상세 HOM001-0500` |
| 화면 유형 | 모달 (오버레이) |
| 진입 경로 | 홈 `내 할 일`의 질문함 섹션 클릭 |
| 관련 기능 ID | `HOME-ACT-01, COMMON-FDB-01` |

## 공통 구조

```
┌────────────────────────────────────────────────────────────┐
│ 질문함 상세                                          [X] │
├────────────────────────────────────────────────────────────┤
│ [전체 N] [새 질문 N] [새 답글 N]                           │
├────────────────────────────────────────────────────────────┤
│ 유형/읽음 │ 질문 제목 │ 스터디 │ 작성자 │ 작성 시각 │ 액션 │
│ ... 세로 스크롤 목록 ...                                   │
└────────────────────────────────────────────────────────────┘
```

- `새 질문`: 참여 중인 진행 스터디에서 팀원이 작성한 질문. 본인 질문은 제외한다.
- `새 답글`: 로그인 사용자의 질문에 팀원이 작성한 답글.
- 알림 이벤트 1건을 1행으로 표시한다. 같은 질문에 답글 3개가 달리면 질문 제목이 같은 3개 행으로 표시한다.
- 행에는 답글 상세 내용/본문 미리보기를 표시하지 않고 질문 제목을 표시한다.
- 질문함은 답글의 답글(대댓글)을 지원하지 않는다.

## 읽음 및 정리 정책

- 행 또는 CTA로 질문을 열면 해당 질문에 연결된 모든 새 질문/새 답글 알림을 읽음 처리한다.
- 읽은 행은 즉시 제거하지 않고 시각적으로 흐리게 표시한다.
- 매일 KST 00:00에 읽음 행을 목록에서 제거한다.
- 읽지 않은 행은 날짜가 바뀌어도 유지한다.
- 홈의 `질문 N · 답글 N`과 우측 합계는 읽지 않은 건만 집계한다.
- 모달 필터 숫자는 00:00 정리 전까지 읽음·읽지 않음 전체를 집계한다.
- 선택 필터에 항목이 없으면 목록 본문에 `내용 없음`을 표시한다.

# 파생 화면 5. 리포트 상세 모달

## 개요

| 항목 | 내용 |

| 화면 ID | `HOM001-0600` |
| Figma 프레임 | `02-11. 리포트 상세 HOM001-0600` |
| 화면 유형 | 모달 (오버레이) |

| 관련 기능 ID | `HOME-ACT-01, COMMON-FDB-01` |

## 목록 정책

- 참여 중인 진행 스터디를 1행씩 표시한다.
- 각 스터디에서 가장 최근 생성된 리포트 1건만 제공한다. 리포트 생성 시점은 스터디마다 다를 수 있다.
- 리포트 생성 완료 스터디를 먼저 배치하고, 완료 스터디끼리는 생성일 최신순으로 정렬한다.
- 스터디명 옆에 별도 주차 배지를 표시하지 않는다. 주차는 리포트명에 포함한다.

| 상태 | 행 정보 | CTA |
|---|---|---|
| 생성 완료 | 스터디명, 주차가 포함된 리포트명, 생성일 | `리포트 보기` → 해당 `RPT001` |
| 생성 전 | 스터디명, `생성 전`, 생성일 `-` | 비활성 `리포트 보기` |

- 홈 보조 문구는 `N개 스터디의 최신 리포트`, 우측 숫자는 최신 리포트가 생성된 진행 스터디 수다.
- 참여 중인 진행 스터디가 없으면 목록 본문에 `내용 없음`을 표시한다.
