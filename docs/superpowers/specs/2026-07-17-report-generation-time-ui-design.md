# 리포트 생성 시간 설정 UI 설계

## 1. 목적

스터디장이 설정 화면에서 리포트 생성 시각을 실제로 변경할 수 있도록 `MGT001-0200` 모달에 시·분 드롭다운 입력 UI를 제공한다. 종료 요일과 적용 정책은 기존 합의 `합의_0717_리포트생성시간.md`를 유지한다.

## 2. 범위

- 대상 화면: `03-07. 리포트 생성 시간 MGT001-0200`
- 관련 기능 ID: `STD-MGT-03`
- 이번 변경은 Figma 기능명세 시안에 한정한다.
- 종료 요일, 시작일, 주차 계산 정책은 변경하지 않는다.

## 3. 모달 정보 구조

모달은 아래 순서로 구성한다.

1. 제목 `리포트 생성 시간`과 닫기 버튼
2. 읽기 전용 `종료 요일`
3. 수정 가능한 `생성 시간`
4. 변경 적용 미리보기
5. `취소`, `저장` 버튼

### 3.1 종료 요일

- 값은 `매주 {종료 요일}` 형식으로 표시한다.
- 입력 컨트롤로 표현하지 않고 읽기 전용 영역으로 표시한다.
- 종료 요일 산정 기준과 변경 불가 정책은 사용자 안내문구로 노출하지 않고 Description에만 명시한다.

### 3.2 생성 시간

- 시간과 분을 독립된 드롭다운으로 제공한다.
- 시간 옵션은 `00시`부터 `24시`까지다.
- 분 옵션은 `00분`, `10분`, `20분`, `30분`, `40분`, `50분`이다.
- 두 드롭다운 사이에는 콜론(`:`)을 표시한다.
- `10분 단위`와 `Asia/Seoul` 기준은 사용자 안내문구로 노출하지 않고 Description에만 명시한다.

### 3.3 24시 처리

- 시간에서 `24시`를 선택하면 분 값을 즉시 `00분`으로 바꾼다.
- `24시`가 선택된 동안 분 드롭다운은 비활성화한다.
- 시간 값을 `00시–23시`로 다시 바꾸면 분 드롭다운을 활성화한다.
- 화면에는 `24:00`으로 표시하고 시스템 해석은 다음 날 `00:00`으로 유지한다.

## 4. 초기값과 적용 미리보기

### 4.1 초기값

- 적용 대기 중인 변경이 없으면 현재 적용 중인 생성 시각을 드롭다운 초기값으로 사용한다.
- 적용 대기 중인 변경이 있으면 마지막으로 저장된 예약 시각을 초기값으로 사용한다.

### 4.2 스터디 시작 전

- 안내 영역 제목은 `적용 안내`로 표시한다.
- 안내문구는 `설정한 시간은 1주차 리포트부터 적용돼요.`로 표시한다.
- 별도의 날짜, 다음 리포트 예정 일시, 시간대 안내는 표시하지 않는다.

### 4.3 스터디 시작 후

- 안내 영역 제목은 `적용 안내`로 표시한다.
- 첫 문장은 `변경한 시간은 다음 주차 리포트부터 적용돼요.`로 표시한다.
- 둘째 문장은 `이번 주 리포트는 기존 설정 시간인 {요일} HH:mm에 생성돼요.`로 표시한다.
- 별도의 주차 레이블, 다음 리포트 예정 날짜, 시간대 안내는 표시하지 않는다.
- 이미 생성된 리포트가 변경되지 않는 정책은 Description에만 유지한다.

## 5. 버튼과 상태

### 5.1 저장 버튼

- 선택값이 모달을 열었을 때의 초기값과 같으면 비활성화한다.
- 시간 또는 분이 달라지면 활성화한다.
- 저장 중에는 중복 제출을 막고 로딩 상태를 표시한다.
- 저장 성공 시 모달을 닫고 적용 시작 주차를 토스트로 안내한다.

### 5.2 취소와 닫기

- `취소` 또는 닫기 버튼을 누르면 변경을 저장하지 않고 모달을 닫는다.
- 저장하지 않은 선택값은 폐기한다.

### 5.3 저장 실패

- 모달과 선택값을 유지한다.
- 버튼 로딩 상태를 해제한다.
- `리포트 생성 시간을 저장하지 못했습니다. 다시 시도해주세요.` 토스트를 표시한다.

## 6. Figma 표현 계약

- 기존 정보 나열 영역을 실제 시·분 드롭다운 컨트롤로 교체한다.
- 기본 프레임에는 닫힌 시·분 드롭다운을 표시한다. 분 옵션과 `24시` 예외는 Description 2에 명시하며 별도 화면 ID는 추가하지 않는다.
- Description은 다음 네 항목을 유지한다.
  1. 종료 요일 고정과 생성 시각만 수정 가능
  2. 시간 `00–24시`, 분 10분 단위, `Asia/Seoul` 기준, `24시` 선택 시 `00분` 고정
  3. 시작 전은 1주차, 시작 후는 다음 주차부터 적용
  4. 완료 리포트 미변경과 예약값 재수정 정책
- 화면 ID `MGT001-0200`과 기능 ID `STD-MGT-03`을 유지한다.
- PM 와이어프레임의 기존 색상, 타이포그래피, 콜아웃 패턴을 재사용한다.

## 7. 접근성·조작 계약

- 시간 드롭다운에서 방향키로 옵션을 이동할 수 있어야 한다.
- 분 드롭다운에서도 방향키로 10분 단위 옵션을 이동할 수 있어야 한다.
- 비활성 분 드롭다운은 시각적으로 비활성 상태가 분명해야 한다.
- 포커스 상태는 테두리 또는 포커스 링으로 구분한다.
- 각 드롭다운의 레이블은 `시간`, `분`으로 식별 가능해야 한다.

## 8. 승인된 설계 결정

- 입력 방식: 시·분 드롭다운
- 분 단위: 10분
- 종료 요일: 읽기 전용
- `24시` 선택: `00분` 자동 고정 및 분 드롭다운 비활성화
- 사용자 안내문구: 적용 시점만 설명형 문장으로 제공
- 종료 요일 산정 기준, 10분 단위, 시간대, 완료 리포트 정책: Description에만 제공
